# Přispívání (Contributing)

Děkujeme za zájem o přispívání do projektu AI Smart Humidor! 🎉

## Jak přispívat

### Hlášení chyb (Bug Reports)
Pokud najdete chybu, prosím:
1. Ověřte, že bug už není nahlášen v [Issues](https://github.com/arisid/humidor/issues)
2. Otevřete nový issue s jasným názvem a popisem
3. Připojte relevantní logy a hardware specifikaci

### Návrhy vylepšení (Feature Requests)
Máte nápad na vylepšení?
1. Otevřete issue se štítkem `enhancement`
2. Popište, co chcete přidat a proč
3. Poskytněte příklady, pokud je to možné

### Přispívání kódu

#### Setup
```bash
git clone https://github.com/arisid/humidor.git
cd humidor
```

#### Postup
1. **Fork** repozitář
2. **Vytvořte feature branch**: `git checkout -b feature/moje-funkcnost`
3. **Commitujte změny**: `git commit -am 'Přidej novou funkcnost'`
4. **Pushněte branch**: `git push origin feature/moje-funkcnost`
5. **Otevřete Pull Request**

#### Pravidla
- Napište jasné commit zprávy (čeština nebo angličtina)
- Aktualizujte README.md, pokud je relevantní
- Testujte změny na vaší hardware konfiguraci
- Respektujte existující code style

### Dokumentace
Pokud najdete nedostatky v dokumentaci:
- Otevřete issue nebo PR s vylepšením
- Aktualizujte README.md, CONTRIBUTING.md nebo komentáře v kódu

## Pravidla chování (Code of Conduct)
- Buďte zdvořilí a respektující vůči ostatním
- Konstruktivní zpětná vazba
- Bez diskriminace nebo nadávek

## Otázky?
Otevřete [Discussion](https://github.com/arisid/humidor/discussions) nebo issue s otázkou!
